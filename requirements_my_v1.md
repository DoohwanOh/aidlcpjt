# Requirements Document

## Introduction

본 문서는 원스탑전환 고객동의 서비스의 요구사항을 정의합니다.
원스탑전환 서비스는 초고속인터넷 및 유료방송 가입자가 사업자 변경 시,
변경 후 사업자에게 가입과 기존 서비스 해지를 동시에 신청할 수 있도록
지원하는 제도입니다. 본 서비스는 Ucube로부터 원스탑전환 서비스 요청을
수신한 후, 고객 본인확인 및 동의 절차를 처리하고, 해지신청 결과를
Ucube에 전달하는 구간을 담당합니다.

기술 스택은 CloudFront/WAF(웹페이지 배포 및 웹 공격 방어),
API Gateway(REST API 엔드포인트),
Lambda(서버리스 백엔드)로 구성됩니다.

## Glossary

- **Ucube**: 원스탑전환 서비스의 상위 시스템으로, 서비스 요청 발송 및 해지신청 처리를 담당하는 외부 연동 시스템
- **원스탑전환서비스(One-stop_Switching_Service)**: 초고속인터넷 및 유료방송 가입자가 사업자 변경 시 변경 후 사업자에게 가입과 해지를 동시에 신청하는 제도
- **고객동의서비스(Consent_Service)**: 본 프로젝트에서 구축하는 시스템으로, 고객 본인확인 및 서비스 전환 동의를 처리하는 웹 서비스
- **본인인증모듈(Identity_Verification_Module)**: 고객의 본인확인을 수행하는 더미 기능 모듈
- **콘텐츠관리모듈(Content_Management_Module)**: 서비스 안내 문구, 공지사항 등 페이지 콘텐츠를 관리하는 백엔드 모듈
- **MMS_URL**: Ucube가 고객에게 발송하는 MMS 메시지에 포함된 서비스 접속 URL
- **서비스안내문구(Service_Guide_Text)**: 원스탑전환 서비스에 대한 안내 내용으로, 버전 관리가 적용되는 텍스트 콘텐츠
- **환경설정(Environment_Config)**: 테스트환경과 운영환경을 분리하여 관리하는 설정 정보

## Requirements

### Requirement 1: Ucube 원스탑전환 서비스 요청 수신

**User Story:** 고객동의서비스 운영자로서, Ucube로부터 원스탑전환 서비스 요청을 수신하여 고객 동의 프로세스를 시작할 수 있기를 원합니다.

#### Acceptance Criteria

1. WHEN Ucube가 원스탑전환 서비스 요청을 전송하면, THE 고객동의서비스 SHALL 요청 데이터를 검증하고 고객별 고유 세션을 생성한다
2. WHEN Ucube 서비스 요청에 필수 항목(고객 식별정보, 서비스 유형)이 누락되면, THE 고객동의서비스 SHALL 요청을 거부하고 누락 항목을 명시한 오류 응답을 반환한다
3. WHEN 고객별 고유 세션이 생성되면, THE 고객동의서비스 SHALL 해당 세션에 연결된 MMS_URL을 생성한다
4. IF Ucube 서비스 요청의 데이터 형식이 유효하지 않으면, THEN THE 고객동의서비스 SHALL 오류 코드와 상세 메시지를 포함한 응답을 Ucube에 반환한다

### Requirement 2: MMS URL 접속 화면

**User Story:** 고객으로서, MMS에 포함된 URL을 통해 원스탑전환 서비스 페이지에 접속하여 서비스 전환 절차를 진행할 수 있기를 원합니다.

#### Acceptance Criteria

1. WHEN 고객이 MMS_URL을 통해 접속하면, THE 고객동의서비스 SHALL 해당 세션의 유효성을 검증하고 원스탑전환 서비스 안내 화면을 표시한다
2. IF MMS_URL에 포함된 세션 정보가 만료되었으면, THEN THE 고객동의서비스 SHALL 세션 만료 안내 메시지를 표시하고 추가 진행을 차단한다
3. IF MMS_URL에 포함된 세션 정보가 유효하지 않으면, THEN THE 고객동의서비스 SHALL 잘못된 접근 안내 메시지를 표시하고 추가 진행을 차단한다
4. WHILE 고객이 서비스 페이지에 접속 중인 동안, THE 고객동의서비스 SHALL 모바일 및 데스크톱 환경에 적합한 반응형 레이아웃을 제공한다

### Requirement 3: 원스탑전환 서비스 안내 화면

**User Story:** 고객으로서, 원스탑전환 서비스에 대한 상세 안내를 확인하여 서비스 전환 내용을 충분히 이해한 후 동의 여부를 결정할 수 있기를 원합니다.

#### Acceptance Criteria

1. WHEN 고객이 서비스 안내 화면에 진입하면, THE 고객동의서비스 SHALL 콘텐츠관리모듈에서 최신 버전의 서비스안내문구를 조회하여 표시한다
2. THE 고객동의서비스 SHALL 서비스 안내 화면에 원스탑전환 서비스의 개요, 전환 절차, 유의사항을 포함한다
3. WHEN 고객이 서비스 안내 내용을 확인한 후 다음 단계 버튼을 선택하면, THE 고객동의서비스 SHALL 서비스 신청 동의 화면으로 이동한다

### Requirement 4: 원스탑전환 서비스 신청 동의

**User Story:** 고객으로서, 원스탑전환 서비스 신청에 대한 동의를 명시적으로 표시하여 서비스 전환 절차를 진행할 수 있기를 원합니다.

#### Acceptance Criteria

1. WHEN 고객이 동의 화면에 진입하면, THE 고객동의서비스 SHALL 필수 동의 항목과 선택 동의 항목을 구분하여 표시한다
2. WHEN 고객이 모든 필수 동의 항목에 동의하고 신청 버튼을 선택하면, THE 고객동의서비스 SHALL 동의 내역을 저장하고 본인인증 단계로 이동한다
3. IF 고객이 필수 동의 항목 중 하나라도 동의하지 않은 상태에서 신청 버튼을 선택하면, THEN THE 고객동의서비스 SHALL 미동의 항목을 강조 표시하고 필수 동의 안내 메시지를 표시한다
4. WHEN 고객이 동의 철회를 요청하면, THE 고객동의서비스 SHALL 동의 철회 확인 팝업을 표시하고 확인 시 서비스 절차를 종료한다

### Requirement 5: 본인인증 처리

**User Story:** 고객으로서, 본인인증을 통해 서비스 신청자가 본인임을 확인하여 안전하게 서비스 전환을 진행할 수 있기를 원합니다.

#### Acceptance Criteria

1. WHEN 고객이 본인인증 단계에 진입하면, THE 본인인증모듈 SHALL 성명, 생일, 성별, 통신사 입력 필드를 포함한 본인인증 요청 화면을 표시한다
2. WHEN 고객이 본인인증 정보를 입력하고 인증 요청 버튼을 선택하면, THE 본인인증모듈 SHALL 입력된 정보의 형식을 검증한 후 본인인증 처리를 수행한다
3. WHEN 본인인증이 성공하면, THE 본인인증모듈 SHALL 인증 성공 결과를 고객동의서비스에 반환하고 고객동의서비스는 해지신청 단계로 이동한다
4. IF 본인인증이 실패하면, THEN THE 본인인증모듈 SHALL 인증 실패 사유를 포함한 결과를 고객동의서비스에 반환하고 고객동의서비스는 재시도 안내를 표시한다
5. IF 고객이 본인인증 입력 필드에 유효하지 않은 형식의 데이터를 입력하면, THEN THE 본인인증모듈 SHALL 해당 필드에 형식 오류 메시지를 표시한다

### Requirement 6: Ucube 해지신청 처리

**User Story:** 고객동의서비스 운영자로서, 고객의 본인인증 완료 후 Ucube에 해지신청을 전달하여 원스탑전환 절차를 완료할 수 있기를 원합니다.

#### Acceptance Criteria

1. WHEN 고객의 본인인증이 성공적으로 완료되면, THE 고객동의서비스 SHALL 고객 동의 내역과 본인인증 결과를 포함하여 Ucube에 해지신청 요청을 전송한다
2. WHEN Ucube가 해지신청 처리 성공 응답을 반환하면, THE 고객동의서비스 SHALL 해지신청 완료 상태를 저장하고 고객 완료 안내 화면으로 이동한다
3. IF Ucube가 해지신청 처리 실패 응답을 반환하면, THEN THE 고객동의서비스 SHALL 실패 사유를 기록하고 고객에게 처리 실패 안내 메시지를 표시한다
4. IF Ucube 해지신청 요청 전송 중 네트워크 오류가 발생하면, THEN THE 고객동의서비스 SHALL 최대 3회까지 재시도를 수행하고 모든 재시도 실패 시 오류 안내를 표시한다

### Requirement 7: 고객 완료 안내

**User Story:** 고객으로서, 원스탑전환 서비스 신청이 완료되었음을 확인하여 이후 절차를 안내받을 수 있기를 원합니다.

#### Acceptance Criteria

1. WHEN 해지신청 처리가 성공적으로 완료되면, THE 고객동의서비스 SHALL 서비스 전환 신청 완료 안내 메시지를 표시한다
2. THE 고객동의서비스 SHALL 완료 안내 화면에 신청 일시, 서비스 전환 예정 내용을 포함한다
3. WHEN 고객이 완료 안내 화면에서 확인 버튼을 선택하면, THE 고객동의서비스 SHALL 세션을 종료하고 서비스 종료 화면을 표시한다

### Requirement 8: 콘텐츠 관리

**User Story:** 서비스 관리자로서, 서비스 안내 문구와 공지사항을 관리하여 고객에게 최신 정보를 제공할 수 있기를 원합니다.

#### Acceptance Criteria

1. THE 콘텐츠관리모듈 SHALL 서비스안내문구의 생성, 조회, 수정 기능을 제공한다
2. WHEN 서비스 관리자가 서비스안내문구를 수정하면, THE 콘텐츠관리모듈 SHALL 새로운 버전 번호를 부여하고 이전 버전을 보존한다
3. THE 콘텐츠관리모듈 SHALL 고객 공지사항의 생성, 조회, 수정, 삭제 기능을 제공한다
4. WHEN 고객동의서비스가 서비스안내문구를 요청하면, THE 콘텐츠관리모듈 SHALL 활성 상태인 최신 버전의 문구를 반환한다

### Requirement 9: Ucube 연동정보 관리

**User Story:** 서비스 관리자로서, Ucube 연동에 필요한 설정 정보를 관리하여 안정적인 시스템 간 통신을 유지할 수 있기를 원합니다.

#### Acceptance Criteria

1. THE 고객동의서비스 SHALL Ucube 연동에 필요한 엔드포인트 URL, 인증 정보, 타임아웃 설정을 관리한다
2. WHEN 서비스 관리자가 Ucube 연동정보를 변경하면, THE 고객동의서비스 SHALL 변경 이력을 기록하고 변경된 설정을 즉시 적용한다
3. THE 고객동의서비스 SHALL Ucube 연동정보에 포함된 인증 정보를 암호화하여 저장한다

### Requirement 10: 환경 분리 관리

**User Story:** 서비스 관리자로서, 테스트환경과 운영환경을 분리하여 운영 안정성을 확보하고 안전한 테스트를 수행할 수 있기를 원합니다.

#### Acceptance Criteria

1. THE 고객동의서비스 SHALL 테스트환경과 운영환경의 설정을 독립적으로 관리한다
2. THE 고객동의서비스 SHALL 환경별로 Ucube 연동 엔드포인트, API Gateway 설정, Lambda 함수 설정을 분리하여 구성한다
3. IF 테스트환경의 설정 변경이 운영환경에 영향을 미치는 요청이 발생하면, THEN THE 고객동의서비스 SHALL 해당 요청을 차단하고 환경 분리 위반 경고를 기록한다

### Requirement 11: 고객 공지 관리

**User Story:** 서비스 관리자로서, 고객에게 표시할 공지사항을 관리하여 서비스 관련 중요 정보를 전달할 수 있기를 원합니다.

#### Acceptance Criteria

1. WHEN 활성 상태의 공지사항이 존재하면, THE 고객동의서비스 SHALL 고객 접속 시 공지사항을 서비스 안내 화면 상단에 표시한다
2. THE 콘텐츠관리모듈 SHALL 공지사항에 제목, 내용, 게시 시작일, 게시 종료일, 활성 상태를 포함한다
3. WHEN 공지사항의 게시 종료일이 경과하면, THE 콘텐츠관리모듈 SHALL 해당 공지사항의 활성 상태를 비활성으로 자동 변경한다
